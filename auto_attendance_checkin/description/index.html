Auto Attendance Check-in/Check-out

Automatically track employee attendance in Odoo 19. This module creates attendance check-in records when users log in and check-out records when they log out or close their browser. Perfect for accurate time tracking without manual intervention.

Key Features

Auto Check-in on Login

Automatically creates attendance check-in record when user logs into Odoo. No manual action required - perfect for seamless time tracking.

Auto Check-out on Logout

Automatically creates check-out record when user logs out. Ensures accurate time tracking for all employee sessions.

Browser Close Detection

Smart session tracking detects when browser is closed without logout. Automatically creates check-out after 15 minutes of inactivity.

Real-time Activity Tracking

Monitors user activity on every request to detect active sessions. Provides accurate login duration tracking and session management.

Duplicate Prevention

Prevents duplicate check-ins if employee is already checked in. Intelligent detection ensures data accuracy and prevents errors.

Employee Linking

Automatically links attendance records to employee based on logged-in user. Only creates attendance if user has linked employee record.

How to Use

01

Ensure that HR Attendance module is installed. Go to Apps menu and search for "Attendances" - make sure it's installed and active.

02

Install Auto Attendance Check-in/Check-out module. Go to Apps → search "Auto Attendance" → click Install button.

03

Link each user to an employee record. Go to Employees → Employees → Open employee record → Set Related User field to the user account → Save.

04

Test the module by logging out and logging back in. When user logs in, attendance check-in record will be automatically created with current timestamp.

05

View attendance records in Attendances → Management menu. You will see all automatically created check-in and check-out records with timestamps.

06

Test check-out by logging out. When user clicks logout, check-out record will be automatically created with current timestamp, completing the attendance session.

07

Test browser close detection. If user closes browser directly without logout, cron job (runs every 5 minutes) will detect inactive session and create check-out after 15 minutes of no activity.

08

Monitor session activity in Settings → Technical → Database Structure → Models → User Session Tracker to view active sessions and last activity timestamps.

09

View comprehensive attendance dashboard in Attendances → Overview → Dashboard to see attendance statistics, worked hours, and attendance history for all employees.

How It Works

Login Flow

  1. User enters credentials and logs into Odoo
  2. Module detects successful authentication
  3. Finds employee record linked to the user
  4. Checks if employee already has active check-in
  5. Creates new attendance record with check-in timestamp
  6. Creates session tracker for activity monitoring

Logout Flow

  1. User clicks logout button
  2. Module intercepts logout request
  3. Finds latest attendance without check-out
  4. Updates attendance record with check-out timestamp
  5. Deactivates session tracker
  6. Completes logout process

Browser Close Detection

  1. Module tracks user activity on every HTTP request
  2. Updates last activity timestamp in session tracker
  3. Cron job runs every 5 minutes
  4. Checks for sessions with no activity for 15+ minutes
  5. Automatically creates check-out for inactive sessions
  6. Cleans up old inactive session records
